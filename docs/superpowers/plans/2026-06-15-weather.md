# V2.5 — Plan 9 (Cycle A): Weather + pressure widget

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development. No *.spec.ts (project decision — verify via build + curl + headless Chrome).

**Goal:** A weather card (current temp/condition, pressure in mmHg+hPa with trend arrow, wind, humidity, precipitation + 7-day mini-forecast) on each water-detail page, fed server-side from the free Open-Meteo API (no key) via a cached service that Cycles B/C will reuse.

**Architecture:** New `OpenMeteoService` (native fetch + AbortController + 30-min in-memory TTL cache keyed by rounded coords) + `WeatherService`/controller exposing `GET /api/weather`. Graceful degradation: on any upstream failure the API returns `{available:false}` and the UI shows a neutral message. Web renders the card SSR (transfer cache).

**Tech Stack:** NestJS (native fetch), Angular 20 SSR, @fishing/shared types.

**Spec:** docs/superpowers/specs/2026-06-15-weather-bite-nearby-design.md (Спільна основа + Цикл A). Dev servers: api :3000, web :4201, Postgres :5433, admin admin/admin12345. Patterns: src/seo/seo.service.ts (config inject), waters module structure, water-detail.ts (SSR + signals + applySeo), common/lang-query.dto.ts, i18n uk/en, styles.scss tokens.

## Task 1: OpenMeteoService (shared fetch+cache) + WeatherService/controller + DTO

**Files:** packages/shared/src/index.ts (WeatherDto, WeatherDailyDto, PressureTrend, WindDir types), apps/api/src/weather/open-meteo.service.ts (shared client+cache), apps/api/src/weather/weather.service.ts (maps raw → WeatherDto + pressure trend + windDir + mmHg), apps/api/src/weather/weather.controller.ts (GET /api/weather), apps/api/src/weather/dto/coords-query.dto.ts (lat/lng validated), apps/api/src/weather/weather.module.ts, app.module.ts (register WeatherModule; export OpenMeteoService from WeatherModule for Cycle B reuse).

- Shared types:
```ts
export type PressureTrend = 'rising' | 'falling' | 'steady';
export type WindDir = 'N' | 'NE' | 'E' | 'SE' | 'S' | 'SW' | 'W' | 'NW';
export interface WeatherCurrentDto {
  tempC: number; feelsC: number; humidity: number; precipMm: number;
  weatherCode: number; pressureHpa: number; pressureMmHg: number;
  pressureTrend: PressureTrend; windKmh: number; windDeg: number; windDir: WindDir;
}
export interface WeatherDailyDto { date: string; weatherCode: number; tMax: number; tMin: number; sunrise: string; sunset: string; }
export interface WeatherDto { available: boolean; current: WeatherCurrentDto | null; daily: WeatherDailyDto[]; updatedAt: string | null; }
```
Rebuild shared.

- open-meteo.service.ts: `@Injectable()` with a private `Map<string, { at: number; data: OpenMeteoRaw | null }>`. Method `fetchForecast(lat: number, lng: number): Promise<OpenMeteoRaw | null>`:
  - key = `${lat.toFixed(2)}|${lng.toFixed(2)}`; if cached && Date.now()-at < 30*60*1000 → return cached.data.
  - build URL (exact params from spec Спільна основа). fetch with `AbortSignal.timeout(4000)`. On non-ok or throw → return null (and cache the null with a SHORTER TTL — cache failures for only 60s so we retry soon: store `{at, data:null}` but treat null entries as expired after 60s). Parse JSON to a loosely-typed `OpenMeteoRaw` (define an interface with the fields requested: current{...}, hourly{time:[],surface_pressure:[]}, daily{...arrays...}).
  - Export `OpenMeteoRaw` interface for Cycle B.
- weather.service.ts: `toWeatherDto(raw): WeatherDto`. If raw null → `{available:false, current:null, daily:[], updatedAt:null}`. Else map:
  - current: tempC=round(current.temperature_2m), feelsC=round(apparent_temperature), humidity=round(relative_humidity_2m), precipMm=current.precipitation, weatherCode=current.weather_code, pressureHpa=round(current.surface_pressure), pressureMmHg=round(surface_pressure*0.750062), windKmh=round(wind_speed_10m), windDeg=round(wind_direction_10m), windDir=degToDir(windDeg), pressureTrend=computeTrend(raw.hourly).
  - degToDir(deg): 8-point — `['N','NE','E','SE','S','SW','W','NW'][Math.round(deg/45)%8]`.
  - computeTrend(hourly): find current hour index by matching the nearest hourly.time to now (or just use last 6 entries up to current). Compare avg of last 3 hourly surface_pressure vs the 3 before that. diff>0.8→'rising', <-0.8→'falling', else 'steady'. If insufficient data → 'steady'.
  - daily: map first 7 of daily arrays → {date:daily.time[i], weatherCode:daily.weather_code[i], tMax:round(temperature_2m_max[i]), tMin:round(temperature_2m_min[i]), sunrise:daily.sunrise[i], sunset:daily.sunset[i]}.
  - updatedAt = new Date().toISOString().
  - Expose a method `getWeather(lat,lng)` = toWeatherDto(await openMeteo.fetchForecast(lat,lng)).
- coords-query.dto.ts: `lat` `@Transform(Number) @IsNumber() @Min(-90) @Max(90)`; `lng` `@Transform(Number) @IsNumber() @Min(-180) @Max(180)`. (Reuse for Cycle B/C — put it in apps/api/src/common/coords-query.dto.ts instead of weather/dto so it's shared. Create at common/.)
- weather.controller.ts: `@Controller('weather')` `@Get()` `getWeather(@Query() q: CoordsQueryDto)` → weatherService.getWeather(q.lat,q.lng). (lang not needed — labels are client-side.)
- weather.module.ts: providers [OpenMeteoService, WeatherService], controllers [WeatherController], exports [OpenMeteoService]. Wire in app.module.
