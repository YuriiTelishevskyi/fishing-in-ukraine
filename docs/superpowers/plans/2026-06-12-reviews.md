# V2.2 — Plan 6: Anonymous reviews with premoderation

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development. No *.spec.ts (project decision).

**Goal:** Visitors leave a review (name + 1–5 stars + text) on a water page without registration; reviews appear publicly only after the owner approves them in the admin. Water cards and detail pages show the aggregated rating; `AggregateRating` lands in JSON-LD.

**Decisions:**
- `Review` model: id uuid, water FK (cascade), authorName (2–40), rating Int 1–5, text (10–2000), status PENDING/PUBLISHED... use PENDING/APPROVED/REJECTED enum `ReviewStatus`; createdAt; `@@index([waterId, status])`, `@@index([status])`.
- Denormalized `ratingAvg Float?` + `ratingCount Int @default(0)` on Water; recomputed in a transaction from the APPROVED set on every moderation transition or review deletion (single source of truth = recompute, never increment).
- Public API: `GET /api/waters/:slug/reviews?lang&page` (APPROVED only, createdAt desc, perPage 10, Paginated<ReviewDto {id, authorName, rating, text, createdAt}> — reviews are NOT localized, raw user text); `POST /api/waters/:slug/reviews` body {authorName, rating, text, website?} — `website` is a HONEYPOT: if non-empty → 400 (silent bot trap); endpoint @Throttle 3/min; creates PENDING; response {ok:true} + message handled client-side. Water must be PUBLISHED → else 404.
- Admin API: `GET /api/admin/reviews?status&page` (default PENDING; includes water {slug, name} for context), `PATCH /api/admin/reviews/:id {status: APPROVED|REJECTED}`, `DELETE /api/admin/reviews/:id`. All recompute aggregates of the affected water.
- Shared DTOs: ReviewDto, WaterListItemDto/WaterDetailDto gain `ratingAvg: number|null`, `ratingCount: number` (mapper passes through).
- Web public (water-detail): «Відгуки» section — summary (big avg + ★ row + count), list (name, date dd.MM.yyyy, stars, text pre-line), form (name input, star radio-group 1–5 with hover state, textarea, hidden honeypot input autocomplete=off tabindex=-1 aria-hidden, submit btn) → on 201 hide form, show success note «Дякуємо! Відгук зʼявиться після модерації.»; errors → inline message. Water card: `★ 4.8 · 12` badge near geo line when ratingCount>0. Detail JSON-LD: add aggregateRating {@type AggregateRating, ratingValue, reviewCount} ONLY when count>0. i18n keys uk/en (reviews.title, reviews.empty, reviews.formTitle, reviews.name, reviews.text, reviews.submit, reviews.thanks, reviews.error).
- Admin UI: nav «Відгуки»; table (filter status default PENDING; columns: Водойма (name+slug link to public page), Автор, ★, Текст (clamped), Дата, Дії: ✓ Схвалити / ✕ Відхилити / 🗑) with optimistic reload; empty state.
- Seed (SEED_DEMO=1): 2 APPROVED reviews on ozero-navariia (different authors/ratings 5 і 4) + aggregates set accordingly; 1 PENDING on stav-murovane (so admin moderation has demo content).
