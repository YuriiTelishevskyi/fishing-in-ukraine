# Catalog MVP — Plan 4 of 4: Production Readiness & Deploy Artifacts

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Harden the API, make SEO origin/404 production-correct, and produce a fully working production Docker stack (Caddy + web SSR + api + Postgres) verified LOCALLY end-to-end, plus a GitHub Actions CI workflow. Actual VPS provisioning/DNS is the owner's manual step, documented in README.

**Architecture:** Multi-stage Dockerfiles for api and web; `docker-compose.prod.yml` with Caddy reverse proxy (routes `/api`, `/uploads`, `/sitemap.xml` → api; everything else → web SSR), named volumes for pgdata/uploads/caddy data; `.env.prod.example`. CI: build + (optional) deploy-over-SSH skeleton, no tests (project decision).

**Hard requirements carried from Plans 1–3 reviews (ALL must land here):**
1. `NODE_ENV=production` pinned for api in prod compose.
2. Rate-limit `POST /api/admin/login` (@nestjs/throttler).
3. helmet on the API.
4. Pin postgres image to a specific minor (e.g. postgres:16.6-alpine) in prod compose.
5. Env-driven SEO origin: replace hardcoded `ORIGIN = 'https://fishmap.ua'` in `seo.service.ts` AND JSON-LD literals with a `SITE_ORIGIN` injection token (server: `process.env['SITE_ORIGIN'] ?? 'https://fishmap.ua'`; browser: read from SSR-transferred state or just keep the token default — simplest: provide in app.config.server.ts like API_BASE and default the browser factory to `'https://fishmap.ua'`).
6. Real 404 HTTP status for the not-found route (SSR server responds 404 when the rendered route is the catch-all).
7. Web SSR host allowlist: configure allowed hosts for the production web container (Angular SSR Host-header protection) — env-driven.
8. Orphaned media files: when a water is deleted, remove its `uploads/waters/<id>/` directory (API change in AdminWatersService.remove → inject StorageService or fs cleanup; MediaService already removes per-file on media delete).

Commit `feat(api): production hardening — throttler, helmet, orphan media cleanup`.
