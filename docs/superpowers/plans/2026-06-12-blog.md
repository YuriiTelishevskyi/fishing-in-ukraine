# V2.1 — Plan 5: Blog (SEO content engine)

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development. No *.spec.ts (project decision).

**Goal:** DB-backed bilingual blog: owner writes Markdown articles in the admin, public reads them at `/blog` (`/en/blog`) with full SEO (Article JSON-LD, hreflang, sitemap). This is the «контент дасть трафік» engine from the business doc.

**Architecture decisions (extends the approved MVP spec):**
- `Article` model (Prisma): id uuid, slug unique (KMU slugify from title), title, titleEn?, excerpt, excerptEn?, content (Markdown), contentEn?, coverUrl? (3-variant webp convention `-full.webp` like Water media, stored at `uploads/articles/<id>/`), status DRAFT/PUBLISHED, publishedAt? (set on first publish), seoTitle?/seoTitleEn?/seoDescription?/seoDescriptionEn?, createdAt/updatedAt.
- Markdown is rendered and sanitized **on the API** (`marked` + `sanitize-html`): public detail returns `contentHtml` — the frontend binds innerHTML without further trust decisions. Localization with uk fallback like waters.
- Public API: `GET /api/articles?lang&page&perPage` (PUBLISHED, publishedAt desc, list DTO: slug/title/excerpt/coverCardUrl/publishedAt), `GET /api/articles/:slug?lang` (detail + contentHtml + raw En fields for admin reuse NOT needed publicly — admin uses its own endpoints).
- Admin API: CRUD `/api/admin/articles` (list incl. drafts, byId returns RAW markdown + all fields; create/update; delete removes cover dir), `POST /api/admin/articles/:id/cover` (sharp 3 variants), publish via PATCH {status} (sets publishedAt when first PUBLISHED).
- Sitemap: add `/blog` + per-article pairs (uk `/blog/:slug`, en `/en/blog/:slug`) with lastmod.
- Web public: routes uk `blog`,`blog/:articleSlug`; en mirror under /en (same segment 'blog'). List page (cards grid: cover/title/excerpt/date) + article page (hero title, cover, `.prose` typography, Article JSON-LD with datePublished/dateModified, breadcrumbs). Header+footer get nav link «Блог»/“Blog”.
- Admin UI: articles table (status/search), editor form (title, slug shown muted, excerpt, markdown textarea monospace min-height 50vh, EN collapsible block, SEO block, cover upload with preview, save/publish buttons). No live MD preview (YAGNI v1).
- Seed: one demo article (gated SEED_DEMO=1) — «Як вибрати платну водойму: 7 порад» with EN translation, so public pages render in dev.
