# Народні рибні точки (community spots) — дизайн

**Дата:** 2026-06-13
**Статус:** затверджено (усне погодження дизайну)

## Мета

Рибалки додають власні рибні точки на карту (геолокація + коментар + опційне фото),
без реєстрації. Кожна точка потрапляє в адмінську дашборду на премодерацію; після
схвалення показується на карті як окремий «народний» шар. Курований каталог водойм
(`Water`) лишається недоторканим.

## Архітектура

Нова сутність `Spot` — окрема від `Water` і `Review`, той самий паттерн премодерації,
що у відгуків (`PENDING → APPROVED → REJECTED`). Public бачить лише `APPROVED`.

### Модель `Spot` (Prisma)

| Поле | Тип | Примітка |
|---|---|---|
| id | uuid | |
| lat, lng | float | обов'язкові; валідуються в bbox України (lat 44–53, lng 22–41) |
| authorName | string | 2–40 символів |
| authorEmail | string? | опційний, **приватний** — лише в адмінці, ніде публічно не показується |
| title | string? | коротка назва, 3–80 (напр. «Озеро біля Іванкова») |
| comment | string | 10–1000 |
| fishNote | string? | вільний текст «щука, окунь», 0–120 |
| photoUrl | string? | `-full.webp`; 3 варіанти (thumb/card/full) у `uploads/spots/<id>/` |
| status | enum SpotStatus | PENDING / APPROVED / REJECTED, default PENDING |
| createdAt | datetime | |

Індекс `@@index([status])`. Контент НЕ локалізується (як подано; як відгуки).

### API

**Публічні:**
- `GET /api/spots` → `SpotDto[]` тільки APPROVED (id, lat, lng, authorName, title, comment, fishNote, photoThumbUrl, photoCardUrl, createdAt). Без пагінації — масив легких пінів (cap 2000, як `/waters/map`). Приймає `lang` (ігнорується, але whitelisted щоб не ловити 400).
- `POST /api/spots` — multipart: поля {authorName, authorEmail?, title?, comment, fishNote?, lat, lng, website (honeypot)} + опційний файл `photo`. Створює PENDING, повертає `{ok:true}`. Honeypot непорожній → 400. `@Throttle 5/год`. bbox-валідація координат → 400 поза межами. Фото: mime jpeg/png/webp, ≤15 MB, sharp 3 варіанти (UUID-first write-then-row, як media.service).

**Адмінські (AdminGuard):**
- `GET /api/admin/spots?status&page&perPage` — за замовч. PENDING; повертає всі поля incl. authorEmail.
- `PATCH /api/admin/spots/:id {status: APPROVED|REJECTED}`.
- `DELETE /api/admin/spots/:id` — також `storage.deleteDir('spots/<id>')`.

### Shared DTO
`SpotDto` (публічний, без email/status) і `SpotStatus` тип у `@fishing/shared`.

## Публічний UX (на сторінці /karta)

- Кнопка **«+ Додати точку»** в панелі фільтрів. Клік → режим постановки: курсор-хрестик,
  клік по карті ставить тимчасовий зелений пін; кнопка **«📍 Моє місцезнаходження»**
  (Geolocation API, з fallback-повідомленням якщо відмовлено).
- Після постановки пін-а відкривається бічна панель (на мобільному — нижній лист) із
  формою: ім'я, email (опц.), фото (опц., 1 файл, прев'ю), коментар, що ловилось (опц.),
  honeypot-поле (приховане). Координати редагуються повторним кліком до сабміту.
- Submit → `{ok:true}` → панель замінюється на «Дякуємо! Точка з'явиться після перевірки».
  Помилки (400/429) → інлайн-повідомлення.
- **Шар народних точок:** зелені піни (`createMapPin(L, 'accent')`? — ні, окремий
  зелений варіант, щоб відрізнявся від каталогу і від акцентного преміуму) поверх карти,
  окремо від кластера каталогу. Попап: фото-прев'ю (card), title або перший рядок
  коментаря, повний коментар, fishNote, ім'я автора, дата. Текст — лише через
  інтерполяцію `{{ }}` (XSS-захист, як у відгуках), фото через `[src]`.
- i18n keys (uk/en) для кнопок/форми/повідомлень. Контент точок — як подано.
- Головна карта — **без змін** (лишається курований каталог).

## Адмінка — дашборд модерації

Новий розділ **«Точки»** в сайдбарі (паттерн «Відгуки»):
- Таблиця (PrimeNG) з фільтром статусу (за замовч. На модерації), пагінація.
- Колонки: фото-thumb, Заголовок + коментар (clamped), Автор (ім'я + email muted),
  Координати (текст), Дата, Дії: **✓ Схвалити / ✕ Відхилити / 🗑 Видалити** (confirm).
- Розгортання рядка (PrimeNG row expansion) показує міні-карту Leaflet із пін-ом точки —
  щоб бачити локацію перед схваленням.
- Усе під AdminGuard; контент через інтерполяцію (XSS).

## Антиспам / безпека

honeypot-поле `website` (непорожнє → 400); `@Throttle 5/год` на POST; фото ≤15 MB +
mime-allowlist + sharp пере-кодовує (вбиває EXIF/пейлоади); bbox-валідація координат;
автор-email ніколи не виходить у публічний DTO.

## Обсяг (3 задачі, паттерн циклу відгуків)

1. **API:** модель + міграція `add_spots`, публічні GET/POST (multipart+photo+honeypot+
   throttle+bbox), адмін-модерація, storage spots/, seed 1–2 демо-APPROVED + 1 PENDING.
2. **Публічна UI:** кнопка + режим постановки + форма-панель на /karta, шар зелених пінів
   із попапами, i18n, SpotDto в api.service.
3. **Адмін:** розділ «Точки», таблиця модерації з row-expansion міні-картою.

Фінальна задача — sweep + холістичне рев'ю + merge.

## Поза обсягом (YAGNI)

лайки/голоси, окрема публічна сторінка точки, точки на головній карті, редагування
рибалкою після подачі, прив'язка точки до існуючої водойми.
