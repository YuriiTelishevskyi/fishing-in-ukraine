# V2.7 — Plan 11 (Cycle C): Nearby waters (місця поруч)

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development. No *.spec.ts (verify via build + curl + headless Chrome).

**Goal:** A geolocation-driven «Поруч» page (`/poruch`, `/en/nearby`) that lists catalog waters nearest the user (distance in km, sorted ascending), plus a «📍 Поруч» entry in the header and a home CTA. Backed by `GET /api/waters/nearby`.

**Architecture:** New service method on the existing WatersService (haversine over PUBLISHED waters — few hundred rows, computed in app, no PostGIS), a `GET /api/waters/nearby` endpoint returning `WaterListItemDto & {distanceKm}` sorted by distance. Web: a CSR-leaning page that requests geolocation, calls the endpoint, renders water-cards with a distance badge; the water-card gains an optional `distanceKm` input. The page is `noindex` (content depends on the user's location — no stable indexable content).

**Tech Stack:** NestJS + Prisma, Angular 20 SSR, @fishing/shared. Reuse coords-query.dto.ts pattern.

**Spec:** docs/superpowers/specs/2026-06-15-weather-bite-nearby-design.md (Цикл C). Dev servers: api :3000, web :4201, Postgres :5433. Patterns: waters.service.ts (list/buildWhere/LIST_INCLUDE/toListItem), waters.controller.ts (GET routes — note `map` before `:slug`!), fish-page/blog-list (simple public page), water-card.ts (badges), locale.service SEGMENTS, header/home, i18n.

## Task 1: API — /api/waters/nearby (haversine, sorted by distance)

**Files:** packages/shared/src/index.ts (NearbyWaterDto = WaterListItemDto & {distanceKm:number}), apps/api/src/waters/dto/nearby-query.dto.ts (lat/lng/radiusKm/limit + lang), apps/api/src/waters/waters.service.ts (+nearby()), apps/api/src/waters/waters.controller.ts (+GET 'nearby' — BEFORE the ':slug' route), apps/api/src/waters/waters.mapper.ts (reuse toListItem; add a small haversine helper or put it in common).

- Shared: `export interface NearbyWaterDto extends WaterListItemDto { distanceKm: number; }`. Rebuild shared.
- common/haversine.ts (apps/api/src/common/haversine.ts): `haversineKm(lat1,lng1,lat2,lng2): number` — standard formula, R=6371. Export.
- nearby-query.dto.ts: extends the lat/lng validation (reuse CoordsQueryDto fields or a fresh dto): `lat` `@Transform(Number) @IsNumber @Min(-90) @Max(90)`; `lng` `@Transform(Number) @IsNumber @Min(-180) @Max(180)`; `radiusKm` `@IsOptional @Transform(Number) @IsNumber @Min(1) @Max(500)` default 50; `limit` `@IsOptional @Transform(Number) @IsInt @Min(1) @Max(100)` default 30; `lang` optional (extend LangQueryDto or add IsIn(LOCALES)).
- waters.service.ts `nearby(q: NearbyQueryDto): Promise<NearbyWaterDto[]>`:
  - fetch PUBLISHED waters with LIST_INCLUDE (the same include `list()` uses) — all of them (few hundred; `where: { status: 'PUBLISHED' }`).
  - map each → `{ ...toListItem(w, q.lang), distanceKm: round(haversineKm(q.lat,q.lng,w.lat,w.lng),1) }`.
  - filter `distanceKm <= radiusKm`, sort ascending by distanceKm, slice to limit.
  - return.
- waters.controller.ts: add `@Get('nearby') nearby(@Query() q: NearbyQueryDto)` — **place it BEFORE `@Get(':slug')`** (route order matters, like the existing `map` route).
