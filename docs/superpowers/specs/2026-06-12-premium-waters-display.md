# Преміум-водойми: модель і відображення в UI (пропозиція)

**Статус:** пропозиція — чекає підтвердження власника перед реалізацією.

## Операційна модель

Без кабінету власника: **адмін вручну вмикає преміум** після домовленості/оплати поза
платформою (дзвінок, рахунок). Це відповідає поточній моделі «адмін веде каталог сам».

## Дані

- `Water.isPremium: boolean` (default false)
- `Water.premiumUntil: DateTime?` — дата закінчення; запис вважається преміум, коли
  `isPremium && (premiumUntil == null || premiumUntil > now)`. Прострочені автоматично
  «гаснуть» без крон-джоби (перевірка на читанні).
- DTO: `isPremium` (обчислений, активний) у WaterListItemDto/MapPinDto.

## Де і як показується (за пріоритетом цінності для покупця)

1. **Каталог + сторінки областей** — преміум першими (сортування:
   `premium desc → verified desc → createdAt desc`), на картці золотий бейдж
   «⭐ Преміум» (градієнт var(--accent) → #D97706, у лівому верхньому куті обкладинки)
   + тонка золота рамка картки (1px) і трохи сильніша тінь.
2. **Головна** — секція «Перевірені водойми» стає «Рекомендовані»: спершу преміум,
   потім verified (та сама сітка, бейджі як у каталозі).
3. **Карта (/karta і карта на головній)** — преміум-пін у золотому варіанті
   (`createMapPin(L, 'accent')` вже існує) трохи більшого розміру, вищий z-index.
4. **Сторінка водойми** — бейдж «⭐ Преміум» біля назви; у блоці «Інші водойми
   області» преміум першими.
5. **Пошук по рибі (/ryba/...)** — те саме сортування і бейджі.

**Не робимо:** позначення в JSON-LD/SEO (платне розміщення не має змінювати
органічну семантику), банери, окремі сторінки.

## Адмінка

- Форма водойми: секція «Преміум» — toggle + date-picker `premiumUntil`.
- Таблиця водойм: колонка-таг «⭐ до 31.08.2026» (accent) / «—»; фільтр «Преміум».

## Обсяг реалізації

Один цикл: міграція (2 колонки) → сортування+DTO в API → бейджі/сортування на
4 публічних поверхнях → адмін-форма/таблиця. Оцінка: 1 план на 3 задачі.
